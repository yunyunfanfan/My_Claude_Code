# 系统性调试

## 问题描述

$ARGUMENTS

## 调试流程

### 第一步：理解错误
- 错误类型和信息是什么？
- 错误发生在哪个文件/行？
- 错误的调用栈是什么？

### 第二步：定位根因
- 读取报错位置的代码
- 向上追溯调用链
- 检查最近的相关变更：!`git log --oneline -10 2>/dev/null`

### 第三步：复现条件
- 什么情况下触发？
- 是否稳定复现？
- 边界条件是什么？

### 第四步：修复
- 提出 2-3 个可能的修复方案
- 选择最安全、影响最小的方案
- 实施修复

### 第五步：验证
- 确认错误不再出现
- 检查是否引入新问题
- 运行相关测试

## 输出格式

1. **根本原因**：一句话说明
2. **修复方案**：代码变更
3. **预防建议**：如何避免类似问题
