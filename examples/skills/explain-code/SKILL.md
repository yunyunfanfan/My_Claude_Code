# 代码解释

## 目标

$ARGUMENTS

## 解释方式

1. **从类比开始**
   - 将代码比作日常生活中的事物
   - 让非技术人员也能理解核心概念

2. **绘制 ASCII 图表**
   - 展示数据流、结构或关系
   - 使用箭头和框图说明流程

3. **逐步解析**
   - 解释每个关键步骤
   - 说明为什么这样设计
   - 指出重要的设计决策

4. **指出常见误区**
   - 什么是常见的错误理解？
   - 有什么容易踩的坑？

5. **实际应用**
   - 这段代码在什么场景下使用？
   - 有什么替代方案？

保持解释对话化，对复杂概念使用多个类比。
